# Lab:Simulation 설계서
## 반응 속도에 영향을 주는 요인 — 농도 · 온도 · 촉매

> Claude Artifact (React) 기반 인터랙티브 시뮬레이션  
> 대상: 중학교 3학년 ~ 고등학교 1학년 과학 수업

---

## 1. 작업 컨텍스트

### 배경 및 목적

실제 실험실에서 수행하기 어려운 화학 반응 속도 실험을 웹 시뮬레이션으로 구현한다.
학생이 변수(농도, 온도, 촉매)를 직접 조작하며 반응 속도 변화를 시각적으로 확인하고, 정리 질문에 답하며 개념을 정착시키는 것이 목적이다.

### 포함 실험 (3개 모듈)

| 모듈 | 실험명 | 핵심 반응 | 시각적 피드백 |
|------|--------|-----------|--------------|
| M1 | 농도와 반응 속도 | KIO₃ + NaHSO₃ + 녹말 → 청람색 | 색 변화 타이머 |
| M2 | 온도와 반응 속도 | 동일 반응, 온도 조건 변경 | 색 변화 + 온도계 애니메이션 |
| M3 | 과산화 수소 분해 | H₂O₂ → H₂O + O₂ (KI 촉매) | 거품 발생 애니메이션 |

### 입출력 정의

- **입력**: 사용자가 UI 컨트롤(슬라이더, 버튼)로 실험 변수 설정
- **출력**: 반응 애니메이션, 경과 시간, 정리 질문 답변 영역

### 주요 제약조건

- 단일 React 컴포넌트 (Artifact 환경, 외부 파일 없음)
- Tailwind CSS 유틸리티 클래스만 사용 (컴파일러 없음)
- 외부 라이브러리: lucide-react, recharts 사용 가능
- 모바일/데스크탑 반응형
- 과학적 정확성: 수치는 실제 실험 결과 범위에 근거

### 용어 정의

| 용어 | 정의 |
|------|------|
| 반응 시간 | 혼합 시작 ~ 색 변화(또는 거품 발생) 까지의 시간 (초) |
| 기준 조건 | 각 모듈의 default 변수값 (슬라이더 중간값) |
| 반응 진행도 | 0 ~ 1 사이의 내부 상태값, 애니메이션 구동에 사용 |
| 스톱워치 | 실험 기구 옆에 배치된 디지털 타이머. 혼합 시작과 동시에 자동 작동, 반응 완료 시 자동 정지. 학생이 수동 Lap/Reset도 가능 |

### 실험 스케일 (스케일업 적용)

소규모(시험관/마이크로 스케일) 대신 **비커 스케일**로 진행한다. 원본 실험의 부피 비율은 그대로 유지하고 전체 부피만 확대한다.

| 모듈 | 원본 총부피 | 스케일업 총부피 | 확대 배율 | 사용 기구 |
|------|-----------|---------------|---------|---------|
| M1 농도 | 2.5 mL | **100 mL** | ×40 | 100 mL 비커 + 50 mL 비커(혼합용) |
| M2 온도 | 2.5 mL | **100 mL** | ×40 | 100 mL 비커 2개 (수조에 담가 온도 조절) |
| M3 촉매 | — | **H₂O₂ 50 mL × 2** | — | 100 mL 눈금 비커 A/B |

> 농도(mol/L)는 부피 변화와 무관하게 동일하게 유지되므로 반응 시간 수치는 그대로 사용 가능하다. 시각적으로 비커가 충분히 차서 색 변화가 더 뚜렷하게 보이는 것이 스케일업의 목적.

## 2. 워크플로우 정의

### 2-1. 전체 흐름

```
앱 시작
  └─► 모듈 선택 화면 (M1 / M2 / M3 탭)
        └─► 실험 준비 단계 (변수 설정)
              ├─► [M1] 농도 슬라이더 조작
              ├─► [M2] 온도 조건 선택 (얼음물 / 실온 / 뜨거운 물)
              └─► [M3] KI 촉매 추가 여부 토글
              └─► "실험 시작" 버튼 클릭
                    └─► 반응 애니메이션 실행
                          └─► 결과 표시 (반응 시간, 그래프)
                                └─► 정리 질문 제시
                                      └─► 다시 실험 / 다른 모듈로 이동
```

### 2-2. 모듈별 단계 및 분기

#### M1 — 농도와 반응 속도

| 단계 | 처리 방식 | 성공 기준 | 실패 처리 |
|------|----------|----------|----------|
| 농도 설정 (0.01 ~ 0.05 M, 4단계) | 슬라이더 UI | 값이 유효 범위 내 | 기본값으로 리셋 |
| 부피 자동 계산 (전체 100 mL 고정, 비율 유지) | 코드 계산 | 비율 합 = 1 | — |
| 실험 시작 | 버튼 클릭 | — | — |
| 반응 시간 계산 | 코드 (수식 기반) | 결과 > 0 | 최솟값 클램프 |
| 색 변화 애니메이션 | React state + CSS transition | 애니메이션 완료 | 타임아웃 후 강제 완료 |
| 결과 그래프 | recharts | 데이터 4점 표시 | — |
| 정리 질문 표시 | 정적 텍스트 | — | — |

**분기 조건**: 농도가 높을수록 반응 시간 단축 (역비례 관계)

**반응 시간 계산 수식 (근사)**:
```
t = t_base / (c / c_base)
t_base = 15초 (기준: 0.05 M)
```

## 3. 구현 스펙

### 3-1. 단일 컴포넌트 구조

```
LabSimulation (root)
├── TabBar                          # M1 / M2 / M3 탭 전환
├── ExperimentModule (현재 탭)
│   ├── ControlPanel               # 변수 조작 UI
│   │   ├── ConcentrationSlider    # M1 전용
│   │   ├── TemperatureSelector    # M2 전용
│   │   └── CatalystToggle         # M3 전용
│   ├── LabBench                   # 실험 기구 시각화 (비커 스케일)
│   │   ├── BeakerAnimation        # M1, M2 (100 mL 비커, 눈금 표시)
│   │   ├── WaterBathAnimation     # M2 (수조 + 비커)
│   │   ├── CylinderAnimation      # M3 (100 mL 눈금 비커 A/B)
│   │   └── StopwatchWidget        # 비커 옆 디지털 스톱워치 (자동 시작/정지)
│   ├── TimerDisplay               # 경과 시간 카운터 (스톱워치와 동기화)
│   ├── ResultChart                # recharts 그래프
│   └── ReviewQuestions            # 정리 질문 + 답변 입력
└── NavigationBar                  # 이전/다음 실험
```

### 3-2. State 구조

```javascript
// 앱 전역
{
  currentModule: 'M1' | 'M2' | 'M3',
  
  // M1
  concentration: number,          // 0.01 ~ 0.05 (M)
  
  // M2
  tempCondition: 'ice' | 'room' | 'hot',
  
  // M3
  catalystAdded: boolean,
  
  // 공통
  phase: 'setup' | 'running' | 'done',
  elapsedTime: number,
  reactionProgress: number,       // 0 ~ 1
  results: Record<string, number[]>  // 누적 실험 결과
}
```

### 3-3. 과학적 수치 기준

```javascript
// M1: 반응 시간 (초) — KIO₃ 농도별 (비커 100 mL 스케일, 비율 동일)
const M1_REACTION_TIMES = {
  0.0125: 55,  // KIO₃ 40 mL / 총 100 mL
  0.0188: 35,  // KIO₃ 60 mL / 총 100 mL
  0.0250: 22,  // KIO₃ 80 mL / 총 100 mL
  0.0313: 14,  // KIO₃ 100 mL / 총 100 mL
};

// M2: 반응 시간 (초) — 온도별
const M2_REACTION_TIMES = {
  ice: 60,
  room: 15,
  hot: 4,
};

// M3: 거품 발생 속도 배율 — KI 유무
const M3_BUBBLE_RATE = {
  noKI: 1,
  withKI: 8,
};
```

### 3-4. 애니메이션 스펙

| 애니메이션 | 방식 | 트리거 | 핵심 표현 |
|-----------|------|--------|---------|
| 비커 색 변화 (투명 → 청람색) | CSS transition, `background-color` | reactionProgress ≥ 1 | 3초 fade |
| 시험관 온도 표시 | border-color + 내부 색조 변화 | tempCondition 변경 | 1초 transition |
| 거품 상승 | CSS keyframe `bubbleRise` | phase = 'running' | 개수·속도를 catalystAdded에 따라 분기 |
| 타이머 카운트업 | setInterval (100ms) | phase = 'running' | 소수점 1자리 |
| 스톱워치 LCD | 비커 옆 위젯, 7-seg 폰트 느낌 div | 혼합 시작 시 자동 start, 색 변화 완료 시 자동 stop | `00:00.0` 포맷, 정지 시 깜빡임 1회 |
| 그래프 등장 | recharts 기본 애니메이션 | phase = 'done' | isAnimationActive |

### 3-5. 주요 UI 컴포넌트 상세

#### 실험 기구 시각화 방식

- **비커 (M1, M2)**: 100 mL 비커를 SVG로 표현. 눈금(20/40/60/80/100 mL) 표시, 액체 높이는 80% 채움, 색상 변화로 반응 시각화
- **수조 + 비커 (M2)**: 외부 수조(얼음물/실온/온수) 안에 비커가 잠긴 형태. 수조 색상으로 온도 조건 표현
- **눈금 비커 A/B (M3)**: 100 mL 비커 2개 나란히, 거품 파티클 absolute 위치로 상승
- **스톱워치 (공통)**: 비커 우측 옆에 디지털 LCD 스타일 위젯 배치. 검정 배경 + 형광 그린 7-seg 텍스트. 혼합 시작과 동시에 작동, 반응 종료 시 자동 정지하여 최종 시간을 고정 표시

#### 정리 질문 영역

각 모듈 완료 후 하단에 슬라이드 등장. 텍스트 입력 없이 선택형(객관식) 또는 서술 힌트 제공 방식.

| 모듈 | 정리 질문 |
|------|----------|
| M1 | KIO₃ 농도가 높아질수록 반응 시간은 어떻게 변하나요? → 선택지 3개 |
| M2 | 온도와 반응 속도의 관계를 설명하세요 → 단어 힌트 제공 |
| M3 | KI가 H₂O₂ 분해에 미치는 영향을 설명하세요 → 선택지 + 설명 |

## 5. 참고 사항

### 과학적 정확성 범위

- 반응 시간 수치는 실제 아이오딘 시계 반응 실험 결과 범위에 근거한 **근사값**
- 아레니우스 방정식 정확 적용 대신 교육용 단순화 수식 사용
- M3의 KI는 촉매임을 반응 전후 상태에서 명시 (소모되지 않음)

### 확장 가능성 (v2 고려)

- 사용자 직접 값 입력 모드
- 실험 결과 데이터 저장 (localStorage 또는 storage API)
- 퀴즈 점수 집계
